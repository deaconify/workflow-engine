When cleaning up milestone bloat, do NOT propose moving issues to later milestones. ALL issues in the milestone need to be addressed.

**Why:** User explicitly rejected deferral twice. The cleanup is about consolidation (merging micro-issues into fewer actionable ones) and organization (sequencing work), not reducing scope. The user wants all work completed, not pushed to later.

**How to apply:** During milestone cleanup, only propose: (1) merging duplicate/overlapping issues, (2) resequencing execution order, (3) fixing structural problems (missing sub-issue links, unclear scope). Never suggest moving issues to a different milestone.
